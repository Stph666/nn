# CARLA 多传感器自动驾驶仿真平台开发汇报文档

## 一、项目概述

### 1.1 项目背景

随着自动驾驶技术的快速发展，高质量仿真平台和标准化数据集成为算法研发的关键基础设施。**CARLA**（Car Learning to Act）是一款开源的自动驾驶模拟器，基于Unreal Engine构建，能够提供高度逼真的物理仿真环境和丰富的传感器接口。

本项目基于 **CARLA 0.9.15** 版本，构建了一套完整的多传感器融合自动驾驶仿真与数据采集平台，旨在解决以下核心需求：

- **算法验证**：在安全的仿真环境中测试自动驾驶感知、规划、控制算法
- **数据集构建**：采集标准化的多模态数据，用于深度学习模型训练
- **传感器融合**：集成多种传感器数据，验证融合算法的效果
- **快速原型**：支持快速配置和部署，加速算法迭代周期

### 1.2 应用场景

| 应用场景 | 说明 |
|---------|------|
| **端到端驾驶模型训练** | 采集RGB图像与对应控制信号，训练端到端神经网络 |
| **障碍物检测算法验证** | 利用激光雷达和相机数据，验证目标检测与跟踪算法 |
| **车道线检测研究** | 通过YOLOPv2等模型进行实时车道线检测与可视化 |
| **自动驾驶系统集成测试** | 在仿真环境中集成感知-规划-控制全链路，验证系统稳定性 |
| **数据增强与扩充** | 通过批量仿真生成多样化天气、光照、交通条件下的训练数据 |

### 1.3 核心能力

| 能力维度 | 功能描述 |
|---------|---------|
| **多传感器集成** | 支持RGB相机、深度相机、激光雷达、语义分割相机、实例分割相机等9种传感器 |
| **驾驶模式** | 手动驾驶（键盘操控）/ CARLA原生自动驾驶 / 定速巡航三模式切换 |
| **数据采集** | 支持10Hz采样率同步记录图像、控制信号、车辆状态，输出标准化数据集 |
| **实时可视化** | Pygame多窗口显示、鹰眼地图、传感器画面网格布局 |
| **自动避障** | 基于激光雷达的障碍物检测与自动刹车逻辑 |
| **环境配置** | 支持自定义天气、地图、车辆类型，灵活配置仿真场景 |

## 三、问题修复记录
### 1. 初始运行报错修复
- **错误1**：`ModuleNotFoundError: No module named 'carla'`
  - 原因：CARLA Python API未正确加入Python路径，egg文件引用路径错误
  - 解决方案：通过glob匹配CARLA安装目录下的egg文件，自动添加至sys.path；同时支持pip安装carla备用方案

- **错误2**：`AttributeError: 'EgoVehicleController' object has no attribute 'controller'`
  - 原因：车辆控制器类未初始化核心控制属性，setup阶段赋值逻辑缺失
  - 解决方案：在EgoVehicleController类__init__中初始化`self.controller = None`，并在setup_ego_vehicle方法中完成carla.VehicleControl对象赋值

- **错误3**：`RuntimeError: Actor not found`（传感器/车辆销毁阶段）
  - 原因：仿真退出时actor已被销毁，重复调用destroy方法触发异常
  - 解决方案：在cleanup函数中增加try-except捕获RuntimeError，按类型过滤actor并安全销毁

- **错误4**：激光雷达数据解析异常导致程序卡顿
  - 原因：点云数据解析未做异常处理，无效数据触发数组越界
  - 解决方案：在lidar_callback中增加全局异常捕获，筛选有效点云数据（x>0、|y|<2）后再计算最近障碍物距离

### 2. 功能逻辑优化
- 修复自动驾驶模式下手动控制信号残留问题：切换自动驾驶时重置油门/转向/刹车参数
- 解决同步模式下world.tick()调用时机错误导致的仿真卡顿
- 优化传感器数据录制逻辑：按采样率（10Hz）控制数据写入，避免磁盘IO过载
- 完善EagleEyeMap绘制逻辑，解决车辆位置更新不及时导致的地图偏移问题

### 4.4 驾驶控制功能

#### 4.4.1 手动驾驶模式（默认）

| 按键 | 功能 |
|------|------|
| W/↑ | 加速 |
| S/↓ | 刹车 |
| A/← | 左转 |
| D/→ | 右转 |
| 空格 | 手刹 |
| C | 定速巡航切换 |

**自动避障逻辑**：

- **< 3.0m**：紧急刹车（brake=1.0）

- **3.0-6.0m**：减速慢行（throttle=0.2, brake=0.3）

- **6.0-10.0m**：轻微减速（throttle=0.35, brake=0.1）

- **> 10.0m**：正常行驶

#### 4.4.2 自动驾驶模式（P键切换）

- 启用CARLA原生traffic_manager进行自动导航
- 切换时自动重置控制信号，避免冲突
- 支持随时切换回手动模式

#### 4.4.3 定速巡航模式（C键切换）

- 设定巡航速度：30 km/h
- 自动调节油门/刹车维持恒定速度
- 结合车道保持逻辑

**回调函数**（`main.py`）：
```python
def lidar_callback(point_cloud):
    global nearest_obstacle_distance
    # 解析点云数据
    points = np.frombuffer(point_cloud.raw_data, dtype=np.dtype([
        ('x', np.float32), ('y', np.float32), ('z', np.float32),
        ('intensity', np.float32)]))
    
    # 筛选前方障碍物（x > 0 且 |y| < 2 米）
    forward_points = [p for p in points if p['x'] > 0 and abs(p['y']) < 2.0]
    
    if forward_points:
        min_distance = min(p['x'] for p in forward_points)
        nearest_obstacle_distance = min_distance
```

### 8.2 详细解决方案

#### 问题1：仿真启动超时

**现象**：
```bash
timeout waiting for simulator
```

**原因分析**：

- **CARLA服务器未启动**：服务器未启动或未正常运行，导致连接超时

- **端口被占用**：默认端口2000被其他程序占用

- **网络连接问题**：防火墙阻止或网络配置异常

**解决方案**：
1. 确认CARLA服务器已启动，等待窗口完全加载
2. 检查端口占用情况：
```bash
# Windows
netstat -ano | findstr ":2000"

# Linux
lsof -i :2000
```
3. 修改代码中`CarlaEnvironment`的`timeout`参数（默认10s，可适当增加）
4. 确保防火墙未阻止本地连接

---

#### 问题2：传感器画面不显示

**现象**：

- Pygame窗口无传感器画面

- 终端无报错信息

**原因分析**：

- **传感器未启用**：`sensors_dict`中对应传感器的`enabled`状态为`False`

- **传感器位置参数不当**：传感器位置坐标设置错误，导致画面异常

- **Pygame渲染异常**：显示线程未正常运行或渲染出错

**解决方案**：
1. 检查`sensors_dict`中对应传感器的`enabled`状态为`True`
2. 确认传感器位置参数（x/y/z）合理：
```python
# 示例配置
sensors_dict = {
    'rgb_camera': {
        'enabled': True,
        'x': 0.8, 'y': 0.0, 'z': 1.7  # 车头前方位置
    }
}
```
3. 重启Pygame窗口，确保显示线程正常运行

---

#### 问题3：数据录制无文件生成

**现象**：

- 按R键后无dataset目录生成

- 终端显示录制已开始但无文件输出

**原因分析**：

- **录制功能未启用**：`DataRecorder`的`enable_recording`初始状态为`False`，需按R键开启

- **输出目录权限不足**：目标目录无写入权限，导致文件无法生成

- **采样率设置不当**：采样率过高或过低，导致数据未正确写入

**解决方案**：
1. 确认`DataRecorder`的`enable_recording`初始状态为`False`（需按R键开启）
2. 检查输出目录权限，确保有写入权限
3. 确认采样率（默认10Hz）设置合理，等待足够帧数据后查看

---

#### 问题4：自动驾驶模式无响应

**现象**：

- 按P键后车辆未自动行驶

- 终端显示自动驾驶已启用但车辆静止

**原因分析**：

- **traffic_manager未初始化**：CARLA的traffic_manager未正确初始化或配置

- **车辆位置不当**：车辆生成位置不在可行驶道路上

- **地图路网不完整**：使用的地图缺少完整的路网数据

**解决方案**：
1. 确认CARLA traffic_manager已初始化：
```python
traffic_manager = client.get_trafficmanager()
traffic_manager.set_global_distance_to_leading_vehicle(2.0)
```
2. 检查`ego_vehicle.set_autopilot(autopilot_enabled)`调用逻辑
3. 使用带完整路网的地图（推荐Town01、Town10HD）

## 项目总结
本项目基于 CARLA 0.9.15 仿真平台，搭建了一套集多传感器数据采集、车辆控制、实时可视化与数据集录制于一体的自动驾驶仿真系统。项目实现了相机、激光雷达等多种传感器的集成部署，完成手动驾驶与 CARLA 自动驾驶模式一键切换，并依托激光雷达实现前方障碍物检测功能。

系统支持自定义采样率的数据录制，可同步保存图像、车辆控制信号、行驶状态等信息，输出标准化数据集。开发过程中解决了环境对接、代码报错、逻辑冲突等多项问题，同时完善了程序异常捕获与仿真资源自动回收机制，保障系统稳定运行。

整体架构模块化、易扩展，不仅可用于自动驾驶算法调试、感知功能验证，也能快速构建训练数据集。后续可围绕多传感器融合、自动避障、车道检测、性能优化等方向继续迭代，进一步提升平台实用性与智能化水平。
